Instalación desde Moodle

Puedes instalarlo directamente desde el directorio de plugins de Moodle a través del menú Administración del sitio > Plugins > Instalar plugins. Sólo tienes que navegar por el directorio y seleccionar instalar. (Tu sitio Moodle necesita estar registrado).

Descarga el módulo

Puedes descargarlo desde el siguiente enlace en la página web de plugins de Moodle. Después de descargarlo, descomprime el contenido en la carpeta /mod del directorio de Moodle.

Alternativamente, puedes descargarlo desde el repositorio de GitHub.

Instalar en  Moodle

Inicie sesión en su página web de Moodle como administrador y visite la página de administración. Aparecerán todos los plugins a actualizar o instalar, incluyendo el módulo de Transferencia de Cursos. Haga clic en el botón Actualizar base de datos de Moodle para proceder a la instalación.

Configura los valores por defecto

El plugin debe ser instalado en todos los sitios Moodle gestionados por la misma organización.
Todos los sitios Moodle requieren que los servicios web de Moodle estén activados y cada sitio Moodle tiene un token de autenticación. Los servicios se configuran cuando se instala el plugin y en ese momento se crea el token de autenticación.
Los sitios se clasifican como "archivados" y "destino" indicando cuál actúa de origen de cursos y a cuales se pueden enviar. El sitio en el que se están programando las operaciones se le denomina sitio "activo".
El sitio "activo" debe tener el sitio "archivado" configurado como origen para poder solicitar las transferencias. Y el sitio "archivado" debe tener como "destino" el sitio "activo", para autorizar la solicitud de transferencia. Así, cada sitio se comunicará utilizando la url y el token del otro sitio en función del papel que desempeñe, origen o destino.
Se puede establecer el tamaño máximo del curso a transferir, así como el tiempo máximo de conexión entre los sitios Moodle de origen y destino. También es importante establecer un valor único que identifique a los usuarios de ambos sitios, los cuales se recomienda dispongan de una gestión de identidad unificada.
